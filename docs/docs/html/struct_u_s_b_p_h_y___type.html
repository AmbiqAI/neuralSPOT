__IOM uint32_t BF00

[0..0] Bypass squelch trigger point configure in chirp modes , active high, keep the default value is strongly recommended . 1: Bypass squelch trigger point configure in chirp modes , 0: squelch trigger point set to 250mV in chirp modes.

[0..0] Dflop output select signal delay compared with digital clock enable signal 1'b0:3 clocks 1'b1:2 clocks

[0..0] Set IO high-Z state. Active high

[0..0] it0

[0..0] All port z bypass value. 1'b1: bypass enable 1'b0:bypass disable

[0..0] 1: DP/DM will be sampled in HS Tx or Rx state 0: DP/DM will be sampled only in Hs Rx state

[0..0] Enable TX shutdown, active LOW. This bit is only used for debug purpose , nothing to do with the normal operation and signal quality, keeping the default value is strongly recommended.

[0..0] This bitfield is reserved.

[0..0] Disconnect detection block input res load sel. 1'b0: disconnect detection block input res load bypass 1'b1: disconnect detection block input res load enable

[0..0] Digital clock enable bypass 1'b1: digital clock bypass enable 1'b0: digital clock bypass disable

◆ BF10

__IOM uint32_t BF10

[1..0] BG output voltage reference adjust, normally these bits are recommended to be kept as the default values. 00: standard center level around 1.25v output, recommended 01: relative higher output 1x: relative higher output

[1..0] HS receiver bias current tuning. 2'b00 represents the minimum bias current 2'b11 represents the maximum bias current

[1..0] BF10 field description needed.

◆ BF11

__IOM uint32_t BF11

[1..1] Turn off LS/FS differential receiver in suspend mode, active low 1: keep the LS/FS differential receiver , pin fss_rxrcv will toggling according to the DP/DM state 0: turn off the LS/FS differential receiver, pin fss_rxrcv will not toggling according to the DP/DM state

[1..1] PLL bandwidth option 1'b0 default 1'b1 increases the PLL bandwidth

[1..1] Tx power down in suspend state. Active low.

[1..1] This bitfield is reserved.

[1..1] Disconnect squelch and comparator calibration bypass, active high

[1..1] Digital clock enable bypass value 1'b1: digital clock enable 1'b0: digital clock disable

__IOM uint32_t BF20

[2..0] This bitfield is reserved.

[2..0] Manually set the Tx Clock phase select. These bits will tune the HS TX path sample timing between digital and analog inside PHY 3'b000 represents the earliest phase 3'b111 represents the latest phase The delay associated with each step is 256ps

[2..0] Rx enable delay select. 3'b000: 4 clocks (480Mhz clock) 3'b001: 5 clocks 3'b010: 6 clocks 3'b011: 7 clocks 3'b100: 8 clocks 3'b101: 9 clocks 3'b110: 10 clocks 3'b111: 12 clocks
